왜 이 화면에 React-style 방식이 맞나 기본 상태

이 화면은 hero, 드롭 현황, 드롭 백, 추천 영역이 같은 상태를 공유합니다. 그래서 영역마다 DOM을 수동으로 고치기보다 상태 기준으로 새 UI 트리를 만들고 필요한 부분만 patch하는 방식이 더 자연스럽습니다.

한 번에 바뀌는 영역 Hero · 드롭 현황 · 드롭 백 · 추천 영역

멤버 입장 여부 하나만 바뀌어도 여러 카드가 동시에 갱신됩니다.

직접 DOM 대비 이점 상태 한 번으로 전체 UI를 다시 계산

직접 DOM 조작이라면 각 카드의 텍스트, 버튼 상태, 배지, 리스트 순서를 따로 찾아 바꿔야 합니다.
