Newsletter Workflow

실제 발행 툴처럼 보이지만, 안에서는 Virtual DOM이 일합니다.

이 페이지는 바닐라 JavaScript만으로 만든 뉴스레터 에디터입니다. 발행본 DOM을 먼저 Virtual DOM으로 바꾸고, 편집 중 HTML을 다시 읽어 새 Virtual DOM을 만든 뒤, Diff 알고리즘으로 변경점을 계산하고 실제 DOM에는 바뀐 부분만 Patch합니다.

step 1 Published DOM -> VDOM step 2 Editor HTML -> New VDOM step 3 Diff 5 Cases step 4 Patch + History

      발표용 핵심 설명

      • 실제 발행본 DOM을 읽어서 Virtual DOM 트리로 변환합니다.
      • 편집 중인 HTML을 다시 읽어 새로운 Virtual DOM을 생성합니다.
      • Diff 알고리즘이 이전 트리와 새 트리를 비교해 최소 변경만 계산합니다.
      • Patch 단계에서 그 변경만 실제 DOM에 반영하고, 결과 상태를 History에 저장합니다.
      • Undo / Redo를 누르면 특정 Virtual DOM 상태로 이동하고 실제/테스트 영역도 함께 동기화됩니다.

      왜 Virtual DOM을 쓰나?
      
          
      
            실제 DOM을 계속 직접 수정하면 브라우저는 레이아웃 계산과 다시 그리기를 반복할 수 있습니다.
            이 데모는 먼저 메모리 안의 Virtual DOM에서 비교와 계산을 끝낸 뒤, 실제 DOM에는 필요한 최소 변경만 적용하는 방식을 보여줍니다.
