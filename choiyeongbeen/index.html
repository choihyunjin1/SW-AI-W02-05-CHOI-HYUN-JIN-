가상 DOM & 패치 알고리즘 실험실

브라우저 실제 DOM을 가상 DOM으로 변환하고, 이전 상태와 비교(Diff)해 바뀐 부분만 실제 화면에 패치(Patch)하는 과정을 검증하는 교육용 워크벤치입니다.

브라우저 DOM API Document / Window / Node

렌더링 비용 (Rendering Cost) 리플로우(Reflow) / 리페인트(Repaint)

          이론 1. 브라우저 탐색은 왜 느린가?
          리플로우(Reflow)와 리페인트(Repaint)의 비용
          
      
            DOM 트리에서 요소 추가, 삭제, 크기 및 속성 변경이 여러 번 일어날수록 브라우저는 전체 화면의 크기나 배치를 모두 재계산하고 화면을 다시 색칠하는 과정을 반복합니다.
            가상 DOM(Virtual DOM)은 이 무거운 비용을 줄이기 위해, 자바스크립트 메모리 상에서 미리 변경점을 단번에 계산해냅니다.
          
      
        
      
        
      
          이론 2. 가상 구조(Virtual DOM)란?
          태그, 속성, 자식 노드를 가진 메모리 객체 트리
          
      
            이 교육용 프로젝트는 실제 물리적인 DOM을 순회(Traverse)하여 가벼운 추상 객체(Object) 모양으로 바꾸고 메모리에 담아 둡니다.
            그 후 오로지 이 가벼운 객체 트리만을 활용하여 빠르고 안전하게 화면의 새로운 모습을 추론합니다.
          
      
        
      
        
      
          이론 3. 핵심 비교(Diff) 분기점 판단
          추가, 삭제, 교체, 속성 변경, 텍스트 업데이트 사항 포착
          
      
            이전의 가상 트리 스냅샷과 새로 만들어낸 가상 트리를 뿌리부터 비교 분석해 위 5가지 케이스에 속하는 변경점만 골라냅니다. 
            그 후 산출된 비교 결과(Patch Log)에 따라, 바뀐 부분과 연결된 물리적 요소 하나만을 족집게처럼 섬세하게 찾아서 업데이트합니다.
          
      
        
      
        
      
          이론 4. 상용 프레임워크(React)와의 연관성
          재조정 (Reconciliation) 이후 커밋 단계 (Commit) 모델
          
      
            현존하는 대표 프레임워크 React 또한 이 앱과 똑같이 상태 변경 후 자신만의 새로운 UI 트리를 우선 구성하고, 이전 트리와 비교하는 '재조정' 과정을 거칩니다.
            이후 바뀐 부분들만을 모아 실제 브라우저 DOM에 '커밋(Commit)'하여 성능을 발휘합니다. 본 시뮬레이터는 이러한 과정을 교육용으로 축소 구현한 것입니다.
