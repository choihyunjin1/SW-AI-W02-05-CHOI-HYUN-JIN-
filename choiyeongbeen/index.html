          왜 실제 DOM이 느린가
          Reflow와 Repaint 비용
          
      
            요소 추가, 삭제, 속성 변경이 많아질수록 브라우저는 레이아웃 재계산과 다시 그리기를 반복합니다.
            Virtual DOM은 이 비용을 줄이기 위해 변경점을 메모리에서 먼저 계산합니다.
          
      
        
      
        
      
          Virtual DOM 구조
          태그, 속성, 자식 노드의 객체 트리
          
      
            이 프로젝트는 실제 DOM을 순회해 element/text 노드를 객체로 바꾸고, 그 트리를 기준으로 화면을
            다시 렌더링합니다.
          
      
        
      
        
      
          Diff 핵심 케이스
          추가, 삭제, 교체, 속성 변경, 텍스트 변경
          
      
            두 Virtual DOM 트리를 비교해 위 다섯 가지 케이스를 찾아내고, 실제 DOM에는 필요한 위치만
            반영합니다.
          
      
        
      
        
      
          React와의 연결
          Reconciliation 이후 Commit
          
      
            React도 상태 변경 후 새로운 UI 트리를 만들고 이전 트리와 비교한 뒤, 바뀐 부분만 실제 DOM에
            commit합니다. 이 프로젝트는 그 축소판입니다.
